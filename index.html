Hash1——为什么是0x61c88647

2015-07-19
Hashing

在Java ThreadLocal类中有一个“奇怪”的数字用于线程变量的hash，它就是0x61c88647。假设存放线程变量的hash表长度为M（通常$M＝2^N$），第K个线程变量在表的位置＝K ＊0x61c88647 ％ M。经过简单地验证，这个hash函数冲突率很低、计算速度很快。本文致力于揭开0x61c88647的神秘面纱。

Hash

作为查找方法的一种，hash就是通过对变元K进行某种算术运算，计算一个函数h(K)，定位K在表中的位置。最理想的hash函数是$h(k1) \neq h(k2)$，即不同变元K对应不同的位置，相互不冲突。但是，满足这种特征的hash函数非常稀少，我们举两个例子来说明：

  1. 将31个元素的集合映射到41个元素的集合中有$41^{31} \approx 10^{50}$种可能的函数，但它们当中仅仅有$ 41 \times 40 \times … \times 11＝41!/10! \approx 10^{43}$个函数能对每一个变元给出不同的值。即大约一千万个函数当中仅仅有一个是适用的

  2. 生日悖论：在一个23个人的房间里，每个人生日都不重复的概率只有0.4927，即随机把23个变元映射到长度为365的一张表中，不冲突的概率要小于0.5

Fibonacci-黄金比例

定义一个数值：$\phi^{-1} = (\sqrt{5}-1)/2 = 0.6180339887$
定义散列函数：$h(k) = \{k \times \phi^{-1}\}$，其中$\{x\}＝x - \lfloor x \rfloor$

散列函数将整数空间散列到［0，1］空间。如下图所示，这些点被分得很开。事实上，每个新增加的点都落入最大的剩下的区间之一，并且按照黄金比例分割它们。(这个现象最先由J.Oderfeld猜测，并且由S.Swierczkowski证明，FundamentaMath. 46(1958), 187-189)。

定理：设$\theta$是任意无理数。当把点$\{\theta\},\{2\theta\},…,\{n\theta\}$放置在线段[0..1]中时，形成点n+1个线段至多有三种不同的长度，并且，下一点$\{(n+1)\theta\}$将落在最大的现存线段之一中。

Conclusion

在ThreadLocal函数中的HASH_INCREASE: $0x61c88647 = (\sqrt(5) - 1) / 2 \times * 2^{32}$，即将所有整数散列到范围[0, $2^{32}-1$]这个范围当中去。

Reference: The Art of Computer Programming: Volume 3

K-means问题总结

2015-07-19
Clustering

When averaged across all possible situations, every algorithm performs equally well.

作为最常用的聚类算法之一，K-means容易理解、实现简单，尽管它不能保证全局最优解，但大部分情况下，它的聚类效果都能满足人们的需求。然而，K-means并不是聚类界的全能选手，甚至有时候会表现的很糟糕。

使用K-means时，我们默认了它的三个前提假设：

  • the variance of the distribution of each attribute (variable) is spherical;
  • all variables have the same variance;
  • the prior probability for all k clusters are the same, i.e. each cluster has roughly equal number of observations; If any one of these 3 assumptions is violated, then k-means will fail.

以2-D数据为例，来看看当不满足上述假设时，K-means的失效是怎样的。

Broken Assumption: Non-Same Variance

标准的K-means使用欧式距离作为相似度衡量标准，当聚类数据各维度之间方差很大，方差大的维度数据对欧式距离的影响相对较大，导致聚类效果下降。假设上图是在笛卡尔坐标系中，数据在Y轴上的方差远大于X轴上的方差（两列数据看起来”很高“），可以想像，K-means hold不住这类分布的数据。

Conclusion

本文从K-means的假设出发，分析了三种不适合使用K-means的情况。然而，这并不代表K-means在这三种情况下就完全不能使用。我们的目标是使用K-means时，应深刻地认识它的原理、假设条件，以此来更好的帮助我们预处理聚类数据（坐标变换、各维度归一化等），让它表现地更好。

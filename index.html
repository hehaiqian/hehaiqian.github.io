how-google-works

2015-11-22

最近拜读了Eric Schmidt写的《How Google Works》，金句良言，收益颇丰。在这个饮酒赏雪煨红炉的日子，决定摘录一些自己共鸣的句子，警示自己。

  1. 随着时间的推移，我意识到，实际上，要想点燃团队的野心是极其困难的。大多数人并没有接触过这种如登月般异想天开的思维方式，他们习惯用“不可能”来否定自己的想法，而不是从基本的物理原理出发去探索可能性。——拉里.佩奇

  2. 不少企业，只求渐变，不求突破。如果只求渐变，时间一长，企业就会逐渐落伍，科技行业尤其如此，因为外界改变通常是革命性的，而不是循序渐进的。所以，你需要强迫自己着眼于未来。希望你们能接受这些想法，也向不可能发出挑战。——拉里.佩奇

  3. “你见过哪个团队的表现能超越既定目标？你的团队研发过比计划中更出色的产品吗？如果答案是否定的，计划还有什么意义，它只是在拖我们的后退罢了”——谢尔盖和拉里曾说过，遇到问题时莫忘“去和工程师谈谈”

  4. 我们如何才能秉承他们的理念，同时创建一家足以对抗世界上最强高科技帝国的公司，实现我们改变全球十几亿人生活方式的宏大理想呢？我们之所以没有在计划中加入这些内容，原因很简单：具体的方法，我们自己也不知道。有关管理的策略，我们唯一确定的是：我们中20世纪所学的东西有一大部分都是错误的，现在到了颠覆过去、重新开始的时候了

  5. 我们帮助谷歌创始人和同人们打造了一家令人称奇的企业，收获了第一手的经验。可以说，我们坐在第一排，目睹了“魔术师”的魔法

  6. 办公室的设计应本着激发活力、鼓励交流的理念，而不要一味的制造阻隔、强调地位

  7. 如果我们手上有数据，那就依数据行事。如果大家只有想法而没有数据，那就按我的想法办

  8. 作为管理者，一旦在团队中发现害群之马，最好的方法是减少其职责，并让骑士接手剩下的工作。作为领导者，你需要给员工以自由和责任，只需让员工知道他们需要对自己的工作负全责

  9. 我们不知道读者从事什么事业，所处何种行业，因此我们不会越俎代庖地指导你如何制定商业计划。但我们敢百分百地打包票：如果你有商业计划，那你的计划一定是错误的。成功的团队懂得如何发现计划中的瑕疵，并及时做出调整

  10. 我没有特殊的禀赋，只是抱有狂热的好奇心————爱因斯坦

  11. 在你的职业生涯初期，能够获得的员工优先认购股权是比较受限的。因此，在正确的行业磨练技能要比在某家公司赌上自己的命运更加合算。在此之后，随着经验的积累，挑选合适的浪头变的越发重要。因为到了那时，股票在你的薪酬构成中所占的比例大幅上升，因此你也应相应地将对公司的挑选放在优先位置

  12. 如果人人想的都一样，那就是有人没有思考————巴顿

  13. 在生活中，很多情况下一件事情需要重复大约20遍才能被人认真听进去。因此，作为领导者，你必须习惯于苦口婆心、诲人不倦————祷文不会因重复而失色

  14. 混乱才是互联网时代中企业的常态。如果万事都看起来似尽在掌握，那只能说明你的速度不够快。当你处于混乱之中的时候，要想把事情做成，唯一的途径就是靠建立关系

  15. 三周原则：在接手新职位的前三周，你不必做什么。你只需要听取大家的心声，看他们的问题和关注点在哪里，了解他们、关心他们、赢得他们的信任

Hash2 — Min Hash（min-wise independent permutation）及其实现

2015-11-22
Hashing

Jaccard similarity

假设我们有两个64-bit的整数集合A和B（其实任何类型的集合都可以通过hash函数变换为整数集合），我们用Jaccard距离来度量两个集合的相似性：
\begin{equation}
J(A, B) = \frac{A \bigcap B}{A \bigcup B}
\end{equation}
对于两个不相交的集合，Jaccard相似度为0；对于两个相同的集合，Jaccard相似度为1。

我们的目标是不用相似函数定义中的交集、并集的操作（慢），快速计算J(A,B)。一个典型的场景：已知一亿个集合，当新来一个集合X时，如何快速判断X与已知集合的相似度是否超过阈值T（搜索引擎每爬取一个新网页都需要判断这个新页面是否与某个老页面相似，以免重复索引）。

设h是一个hash函数，它可以将所有64-bit的数字全排列（全排列hash是指：所有数字被h映射到一个新的位置，不同数字的位置不同）。对于集合S，$h_{min}(S)$表示集合S经过h映射后的最小值。现在我们将$h_{min}$作用于集合A和B，得到
\begin{equation}
P(h_{min}(A) = h_{min}(B)) = J(A,B)
\end{equation}
这个等式可以这样理解：$h_{min}(A)$和$h_{min}(B)$的所有元素都是由集合${A \bigcup B}$中的元素映射而来，其中满足$h_{min}(A) = h_{min}(B)$条件的所有元素则都是由集合$A \bigcap B$中的元素映射而来。

Hash1 — 为什么是0x61c88647

2015-07-25
Hashing
  1. 0x61c88647 = 1640531527
  2. $2^{31} * (\sqrt(5) - 1)/2 = 2654435769$, as 32 bit unsigned
  3. $2^{31} * (\sqrt(5) - 1)/2 = -1640531527$, as 32 bit signed

在Java ThreadLocal类中有一个“奇怪”的数字用于线程变量的hash，它就是0x61c88647。假设存放线程变量的hash表长度为M（通常$M＝2^N$），第K个线程变量在表的位置＝K ＊0x61c88647 ％ M。经过简单地验证，这个hash函数冲突率很低、计算速度很快。本文致力于揭开0x61c88647的神秘面纱。

Hash

作为查找方法的一种，hash就是通过对变元K进行某种算术运算，计算一个函数h(K)，定位K在表中的位置。最理想的hash函数是if $k1 \neq k2$, then $h(k1) \neq h(k2)$，即不同变元K对应不同的位置，相互不冲突。但是，满足这种特征的hash函数非常稀少，我们举两个例子来说明：

  1. 将31个元素的集合映射到41个元素的集合中有$41^{31} \approx 10^{50}$种可能的函数，但它们当中仅仅有$ 41 \times 40 \times … \times 11＝41!/10! \approx 10^{43}$个函数能对每一个变元给出不同的值。即大约一千万个函数当中仅仅有一个是适用的

  2. 生日悖论：在一个23个人的房间里，每个人生日都不重复的概率只有0.4927，即随机把23个变元映射到长度为365的一张表中，不冲突的概率要小于0.5

Fibonacci-黄金比例

定义一个数值：$\phi^{-1} = (\sqrt{5}-1)/2 = 0.6180339887$
定义散列函数：$h(k) = \{k \times \phi^{-1}\}$，其中$\{x\}＝x - \lfloor x \rfloor$

散列函数将整数空间散列到［0，1］空间。如下图所示，这些点被分得很开。事实上，每个新增加的点都落入最大的剩下的区间之一，并且按照黄金比例分割它们。(这个现象最先由J.Oderfeld猜测，并且由S.Swierczkowski证明，FundamentaMath. 46(1958), 187-189)。

定理：设$\theta$是任意无理数。当把点$\{\theta\},\{2\theta\},…,\{n\theta\}$放置在线段[0..1]中时，形成点n+1个线段至多有三种不同的长度，并且，下一点$\{(n+1)\theta\}$将落在最大的现存线段之一中。

Conclusion

在ThreadLocal函数中的HASH_INCREASE = $0x61c88647 = (\sqrt(5) - 1) / 2 \times * 2^{31}$，即将整数尽可能均匀地散列到[0, $2^{32}-1$]这个范围当中去。

Reference: The Art of Computer Programming: Volume 3

K-means问题总结

2015-07-19
Clustering

When averaged across all possible situations, every algorithm performs equally well.

作为最常用的聚类算法之一，K-means容易理解、实现简单，尽管它不能保证全局最优解，但大部分情况下，它的聚类效果都能满足人们的需求。然而，K-means并不是聚类界的全能选手，甚至有时候会表现的很糟糕。

使用K-means时，我们默认了它的三个前提假设：

  • the variance of the distribution of each attribute (variable) is spherical;
  • all variables have the same variance;
  • the prior probability for all k clusters are the same, i.e. each cluster has roughly equal number of observations; If any one of these 3 assumptions is violated, then k-means will fail.

以2-D数据为例，来看看当不满足上述假设时，K-means的失效是怎样的。

Broken Assumption: Non-Same Variance

标准的K-means使用欧式距离作为相似度衡量标准，当聚类数据各维度之间方差很大，方差大的维度数据对欧式距离的影响相对较大，导致聚类效果下降。假设上图是在笛卡尔坐标系中，数据在Y轴上的方差远大于X轴上的方差（两列数据看起来”很高“），可以想像，K-means hold不住这类分布的数据。

Conclusion

本文从K-means的假设出发，分析了三种不适合使用K-means的情况。然而，这并不代表K-means在这三种情况下就完全不能使用。我们的目标是使用K-means时，应深刻地认识它的原理、假设条件，以此来更好的帮助我们预处理聚类数据（坐标变换、各维度归一化等），让它表现地更好。
